# Vanguard Dear Days Research Notes

Date: 2026-06-19

Scope: local, non-invasive inspection of the installed Steam folder at
`C:\Program Files (x86)\Steam\steamapps\common\VGDD`.

This note is for architecture inspiration only. Do not copy proprietary code,
assets, extracted data tables, textures, card art, or packaged resources from
Vanguard Dear Days.

## Safe Inspection Boundary

- Allowed: folder inventory, executable metadata, readable string names, save
  folder structure, high-level architecture inference.
- Not allowed: DRM bypass, decrypting protected content, unpacking assets for
  reuse, decompiling implementation code, or shipping anything copied from the
  game.

- Data package:
  - `app/data.lpkg`
  - `app/data.lpkg.0` through `app/data.lpkg.18`

- The executable contains strings such as:
  - `Cardfight!! Vanguard Dear Days`
  - `ver.1.7.1`
  - `LcFramework`
  - `LcThreadPool`
  - `DirectX`
  - `D3D11`
  - `data.lpkg`

- Save path seen in strings and on disk:
  - `%USERPROFILE%/Saved Games/VG2/SAVELOAD`

Inference: the game appears to use a custom Lancarse/FURYU-style framework and
LPKG package format, not a normal Unity `*_Data` layout. This does not change
our Unity decision; it mainly confirms that Dear Days is a dedicated rule engine,
not a lightweight manual tabletop layer.

runtime; it chooses among legal actions generated by rule and ability systems.

## Implications For Our Project

Use this architecture direction:

```text
Card database text
  -> structured ability definitions
  -> event/trigger matcher
  -> cost and target resolver
  -> primitive effects
  -> legal action generator
  -> bot policy chooses legal actions
  -> reducer/event log applies actions
```

The CPU should be downstream of the rule engine. Do not make the CPU mutate
state directly or carry card-specific rule hacks that bypass the core action
model.

## Recommended Next Milestone Insert

Before making the CPU smarter, add an Ability Core milestone:

1. Ability schema for `AUTO`, `ACT/Startup`, `CONT`, `ORDER`, and trigger
   abilities.
2. Event hooks such as ride, place, attack declared, boost, guard, drive check,
   damage check, battle end, turn end.
3. Cost primitives: counter blast, soul blast, discard, rest, retire, bind.
4. Target primitives: self, chosen unit, chosen circle, deck search, drop,
   damage, soul, opponent unit.
5. Effect primitives: draw, power, critical, call, move, retire, bind, search,
   reveal, shuffle, stand, rest, add marker, add quick shield.
6. Continuous ability recalculation pass.
7. Manual fallback for unsupported card text.
8. Bot integration only through legal actions.

## Design Rule

Treat Dear Days as proof that the correct core is an action plus ability engine.
Do not treat it as a source for copied assets, copied data, or copied code.
