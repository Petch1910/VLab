# UI Icon Override Spec

## Purpose

This spec allows a private user-provided icon pack to override the safe default
icons used by the UI. It exists so the project can support personal asset
replacement without storing, distributing, extracting, or documenting how to rip
proprietary assets from VangPro, Vanguard Area, Dear Days, Dear Days 2, or
official websites.

Trigger overrides are optional. The default UI should remain usable with
English text badges only: `CRITICAL`, `DRAW`, `FRONT`, `HEAL`, `OVER`, and
`STAND`.

## Boundary

Codex must not:

- extract icons from commercial apps or games
- decompile apps to recover UI assets
- scrape official websites for protected UI art
- commit proprietary icons into the repo
- provide steps for bypassing app/game asset protection

Codex may:

- create an override folder and manifest
- load user-provided files by semantic key
- fall back to the safe Lucide/custom symbols when an override is missing
- validate file existence and dimensions
- document the expected filenames and semantic mapping

## Runtime Priority

When UI icon loading is implemented, icons should resolve in this order:

1. `client/unity/VanguardThaiSim/Assets/UI/Icons/UserProvided/`
2. `client/unity/VanguardThaiSim/Assets/UI/Icons/VanguardSymbols/` if created
3. `client/unity/VanguardThaiSim/Assets/UI/Icons/Lucide/`

Missing user-provided files must never block the UI. They should fall back to
the safe default icon and optionally show a validation warning in development.

## User-Provided Folder

Folder:

```text
client/unity/VanguardThaiSim/Assets/UI/Icons/UserProvided/
```

Supported formats for the first pass:

- `.png`
- `.svg` only if Unity import is reliable on the target build

Recommended PNG sizes:

- `64x64`
- `128x128`
- transparent background

## Semantic Keys

The UI should reference semantic keys, not file names scattered through code.

### Trigger Keys

- `trigger_critical`
- `trigger_draw`
- `trigger_front`
- `trigger_heal`
- `trigger_over`
- `trigger_stand_legacy`

### Marker Keys

- `marker_force`
- `marker_accel`
- `marker_protect`
- `marker_quick_shield`
- `marker_persona`
- `marker_crest`
- `marker_energy`
- `marker_gauge`

### Zone Keys

- `zone_deck`
- `zone_hand`
- `zone_drop`
- `zone_damage`
- `zone_soul`
- `zone_bind`
- `zone_order`
- `zone_replay`

## Manifest Shape

Example:

```json
{
  "schema": "vanguard-icon-pack-v1",
  "pack_id": "private_user_icons",
  "display_name": "Private User Icons",
  "icons": {
    "trigger_critical": "trigger_critical.png",
    "trigger_draw": "trigger_draw.png",
    "trigger_front": "trigger_front.png",
    "trigger_heal": "trigger_heal.png",
    "trigger_over": "trigger_over.png",
    "marker_quick_shield": "marker_quick_shield.png",
    "marker_persona": "marker_persona.png"
  }
}
```

## M19 Implementation Status

`M19-09` is implemented as a safe model/validator layer.

- `UiGameSymbolRegistry` exposes default semantic-key symbol definitions.
- `UserIconPackManifest.FromJson()` reads the existing `icons` manifest object.
- `UserIconPackValidator` validates declared files, blocks paths outside the
  user-provided folder, ignores unknown keys, and treats missing files as
  fallback warnings instead of UI blockers.
- Home/Lobby trigger badges now resolve through semantic keys while still using
  safe English text labels by default.
- No proprietary icons are included in the repo.

## Acceptance Criteria

- Done: the repo contains only the override manifest/template and documentation.
- Done: Home/Lobby trigger symbols resolve by semantic key.
- Done: missing private icons fall back cleanly.
- Done: path traversal is rejected.
- Done: no proprietary icons are committed or generated by Codex.
